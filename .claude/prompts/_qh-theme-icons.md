# Задача: на полигоне /dev/req-cards добавить вариант «темы как иконки-only с подписью по наведению/тапу»

Файл полигона: `src/app/dev/req-cards/page.tsx`. Это dev-витрина (noindex), НЕ продакшн-карточка. Меняем только полигон и при необходимости добавляем один новый клиентский компонент-хелпер.

## Что есть уже (используй, не дублируй)
- `src/data/themes.ts` экспортирует `THEMES` — у каждой темы уже есть `slug`, `label` (рус.) и `Icon: LucideIcon`. Бери иконку и подпись отсюда, маппинг не выдумывай.
- В полигоне темы сейчас рендерятся как текстовые чипы (см. блок `interestLabels` в `RequestCard`, строки ~247-255), label берётся из `getInterestLabels` → лейблы тем.
- Компонент тултипа уже есть: `src/components/ui/tooltip.tsx` (shadcn/Radix: `Tooltip`, `TooltipTrigger`, `TooltipContent`, `TooltipProvider`).

## Что сделать
Цель сравнения: показать рядом «темы текстом» (как сейчас) и «темы только иконками, подпись по наведению/тапу».

1. Добавить на страницу новую секцию сравнения ПОД существующими тремя секциями типа группы — отдельный блок с заголовком, например: «4 · Темы: текст vs иконки-only». В ней те же 4 карточки матрицы, но строка тем рендерится в двух трактовках, чтобы было видно различие:
   - подсекция «Текст» — как сейчас (текстовые чипы).
   - подсекция «Иконки-only» — каждая тема = только иконка `THEMES[].Icon` в маленьком чипе, БЕЗ текста; подпись (`label`) показывается тултипом.
2. Иконка-only чип:
   - размер иконки 14px, тон `text-ink-2`, нейтральный серый — не вводить новый цвет (цвет на карте остаётся только у статус-бейджа). Чип компактный круглый (`rounded-full`), как остальные мета-чипы.
   - подпись открывается ПО НАВЕДЕНИЮ (desktop hover) И ПО ТАПУ (mobile). Radix Tooltip по тапу сам не открывается надёжно — сделай чип кнопкой с контролируемым состоянием тултипа: тап переключает `open`, hover открывает как обычно. На 375px подпись ОБЯЗАНА появляться по тапу — это требование SHIP_GATE.
   - доступность: у триггера должен быть `aria-label={label}`, чтобы тема читалась скринридером и без открытия тултипа.
3. Граница client/server (ВАЖНО, AP-014): `page.tsx` — серверный компонент и экспортирует `metadata`; НЕ добавляй `'use client'` в него. Интерактивный тултип-чип вынеси в отдельный новый клиентский компонент-файл с `'use client'` (например `src/app/dev/req-cards/theme-icon-chip.tsx`) и импортируй его в page.tsx. Клиентский файл импортирует `THEMES`/типы только как данные/типы (themes.ts не серверный — это ок), не тащи серверные модули.
4. Не трогай продакшн-карточки и `src/components/shared/req-card.tsx`. Только полигон + новый хелпер.
