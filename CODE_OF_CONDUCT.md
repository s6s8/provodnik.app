# Code of Conduct

## Our Standard

This project expects respectful, direct, and professional collaboration.

Examples of acceptable behavior:

- giving clear technical feedback without personal attacks
- assuming good intent while staying candid about problems
- documenting decisions and tradeoffs so others can follow the work
- respecting privacy and security when handling data or credentials

Examples of unacceptable behavior:

- harassment, discrimination, or intimidation
- publishing private information without permission
- deliberately disruptive behavior in issues, pull requests, or discussions
- bad-faith technical review or repeated personal attacks

## Scope

This code of conduct applies in repository discussions, issue threads, pull requests, and any other project-maintained collaboration channel.

## Enforcement

Project maintainers may remove or moderate comments, issues, pull requests, or contributors that violate this policy.
