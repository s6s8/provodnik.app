# DESIGN_REFACTOR_PLAN — Provodnik

Produced by the full-app design audit of 2026-07-13 (branch `audit/design-refactor-plan-20260713`). Evidence, rubric, inventory and per-route findings live in `.design-audit/` (RUBRIC.md, INVENTORY.md, FINDINGS.md, findings/G1…G6, screens/). North Star: a trustworthy request-first marketplace — travelers post requests, active approved guides make reliable offers.

## 1. Executive summary

1. The app has a real token system (navy/amber/green, radii, shadows in `globals.css`) and a solid Button/Badge/Card kit — the foundation is good.
2. But two core text tokens fail WCAG AA everywhere: muted text is 2.97:1 (199 files) and the amber/green badge tints are 2.5–3.6:1 on every request card.
3. Pages bypass the kit constantly: 432 arbitrary px values, 491 banned `space-y-*`, 42 raw palette classes, 18 hex literals, 18 hand-copies of the glass recipe.
4. The same UI is built 2–4 different ways: 3 request cards, 2 EmptyStates, 2 AvatarStacks, 2 PublicGuideCards, 3 step-card styles, 4 stat-tile recipes, 5 content widths, 3 eyebrow systems, 5 segmented-control styles.
5. Keyboard/AT support is patchy: 3 bespoke input recipes kill focus rings, forms lack `aria-invalid`/`role="alert"` wiring, icon-only buttons lack labels.
6. Mobile has concrete breaks: /trips tabs truncate at 375, ListRow actions overflow by ~58px, header hamburger escapes the pill, admin has NO nav at 768–1023px.
7. Status colors are semantically scrambled (warnings shown as destructive red, contradictory dispute maps one click apart).
8. The fix is 4 waves: W1 token+atom repairs → W2 shared-component consolidation → W3 per-page cleanups → W4 a11y/copy/motion polish; 41 tasks, each standalone.
9. Everything reuses existing tokens/components — no new dependencies, no new abstractions beyond one extracted StatTile and three shadcn CLI atoms (checkbox, switch, toggle-group).
10. Verification is mechanical: contrast script + grep gates + screenshot diffs per wave at 375/768/1440 + the repo verify chain.

## 3. Systemic issues, ranked (pages affected × max severity)

Full detail with file:line in `.design-audit/FINDINGS.md` (SYS-1…SYS-14):

1. **SYS-1/2/3 — AA contrast baked into tokens** (all ~60 pages, P0): muted #8A93A1 = 2.97:1; badge warning/success tints 2.5/3.5:1; white-on-gold 2.88:1; placeholder 2.04:1; footer alphas ~3:1.
2. **SYS-6 — component duplication** (~40 pages, P1): 3 request cards, 2 EmptyStates, 2 AvatarStacks, 2 PublicGuideCards, glass recipe ×18, step cards ×3, stat tiles ×4, content widths ×5, eyebrows ×3, segmented controls ×5, headers ×8 hand-rolled.
3. **SYS-5 — 432 arbitrary values off the scale** (~50 files, P1): text-[10–15px], pt-[54px]×9, gap-[7–11px], min-h-[44px]×20, rounded-[10–28px], z-[50–140].
4. **SYS-4 — space-y ×491 in 121 files** (P1, banned pattern).
5. **SYS-7 — focus-visible gaps** (~30 files, P1): 3 bespoke input recipes, chips/links/accordions without rings, 2 competing ring recipes.
6. **SYS-8 — form a11y wiring** (all forms, P1): no aria-invalid/aria-describedby/role="alert", pending states without spinner, native checkbox/switch hand-rolled.
7. **SYS-10 — mobile breaks** (P1): /trips tab truncation, ListRow 58px overflow, header pill overflow, admin nav hole 768–1023, bottom-nav wrap.
8. **SYS-9 — raw palette/hex leakage** (P1): 42 palette classes (param badges!), 18 hex literals, rgba recipes.
9. **SYS-11 — status semantics scrambled** (P2): warning→destructive, contradictory dispute maps, rose tint on neutral param.
10. **SYS-13 — vocabulary drift** (P2): Provodnik/«Проводник», Мои поездки/запросы, "Найти тур", eyebrow babel, stale dates.
11. **SYS-14 — empty/loading/error inconsistency** (P2): text-only loading ×4, empty-no-action ×8, error-no-retry ×6.
12. **SYS-12 — dead surface** (P2): unreachable .dark block, unused .native-picker-hidden, hero-overlay vs scrim duplication, 5 orphan components.

## 4. Target design-system specification (actual values)

All changes land in `src/app/globals.css` (single source of truth) and the ui atoms. Values below are canonical after W1.

### Color tokens (Layer 1 additions/changes in `:root`)
```css
/* CHANGED */
--muted-ink:   #68727F;   /* was #8A93A1 — 4.6:1 on #FAFAF9, 4.9:1 on #fff */
--faint:       #98A1AE;   /* was #AEB6C2 — placeholder-only */
--gold-foreground: #231505; /* was #fff (2.88:1 on gold) — dark ink on amber */
/* NEW — text-safe status tones (backgrounds keep existing tints) */
--warning-text: var(--amber-600);  /* #8F5817 — 5.5:1 on --amber-tint */
--success-text: var(--green-600);  /* #27764F — 4.8:1 on --green-tint */
/* NEW — decorative-only gray (the old muted), never for text */
--ink-faint:   #8A93A1;
```
`@theme inline` additions: `--color-warning-text: var(--warning-text); --color-success-text: var(--success-text); --color-ink-faint: var(--ink-faint);`

Semantic usage rules: body text `text-foreground`/`text-ink-2`; secondary/meta `text-muted-foreground` (now 4.6:1); decorative only `text-ink-faint`; status text on tints `text-warning-text`/`text-success-text`; footer on navy: alphas ≥ /60.

### Typography
- Scale: `text-xs` 12 / `text-sm` 14 / `text-base` 16 / `text-lg` 18 / `text-xl` 20 / `text-2xl` 24 / `--text-section` clamp(24–30) / `--text-display` clamp(46–68). **No `text-[Npx]` anywhere.**
- Mobile body floor 16px (`text-base`) for reading copy; 14px only for meta; 12px minimum ever.
- Weights: 700/600 headings, 600 buttons/labels-strong, 500 labels, 400 body.
- Eyebrow = `Badge variant="eyebrow"` (12px, uppercase, primary) — the only eyebrow.
- Page title = `PageHeader` (h1 30px) on every workspace page; marketing heroes use InfoHero/ImmersiveHero clamps.

### Spacing & layout
- 4/8 grid; allowed gaps: 1, 2, 3, 4, 6, 8; section padding `pt-14` (56px) / `--sec-pad` 80px; page rhythm `flex flex-col gap-8`.
- Containers: workspaces `max-w-page` (1160px, from protected layout); documents (policies/for-business/info) `max-w-3xl`; auth cards `max-w-sm/md`. Gutter always `px-gutter`.
- `min-h-dvh` replaces `min-h-screen`.
- Touch targets ≥44px (`h-11`/`min-h-11`/`size-11`), ≥8px between targets.

### Radius / shadow / z
- Radii tokens only: `rounded-btn` 10 / `rounded-step` 12 / `rounded-lg|card|glass` 16 / `rounded-hero` 18 / `rounded-full`. No `rounded-[N]`.
- Shadows: `shadow-card`, `shadow-lift`, `shadow-glass` only (hover = `hover:shadow-lift`).
- Z ladder: content 0–10; sticky bars `z-40`; nav `z-100`; popover `z-[110]`; dialog/tooltip `z-[120]` (existing atom values). No other z values.

### Focus & states
- One recipe everywhere (Button's): `outline-none focus-visible:border-ring focus-visible:ring-3 focus-visible:ring-ring/40`.
- Disabled: `disabled:pointer-events-none disabled:opacity-50`. Pending: Button `loading` prop (Spinner + aria-busy) — never text-swap ellipsis.
- Inputs: only `ui/Input`, `ui/Textarea`, `ui/Select`, `ui/Checkbox`, `ui/Switch` (last two added via CLI). Errors: `aria-invalid` on control + message `role="alert"` linked by `aria-describedby`.

### shadcn variant policy
- **Badge** is the only pill: variants default/secondary/destructive/outline/ghost/link/eyebrow/success/warning/info/overlay; base `text-xs`; success/warning use the new text tokens. Tag, interest-tag, hand-rolled pills fold into it (Tag may remain as a thin wrapper mapping to Badge variants or be deleted).
- **Button** gains `variant="success"` (moderation approve). Sizes as-is; `min-h-[44px]` per-usage forbidden — use `size` prop.
- **Card** plain (no `border-border/70 bg-card/90` per-usage tints); glass surfaces use **GlassCard**.
- **EmptyState**: single component (`shared/empty-state`, absorbs ui/ extras), always icon+title+description+action.
- **Skeleton/ListRowSkeleton** for every loading state; no text-only loading, no bare animate-pulse divs.
- **StepCard** for every numbered-steps block; **StatTile** (new, extracted from stats page Card pattern) for every stat row; **StickyActionBar** for every fixed bottom bar; **PageHeader** for every workspace h1.
- Dark theme: tokens stay, documented as reserved-unreachable (no toggle shipped; parity re-audited if/when a toggle lands).
