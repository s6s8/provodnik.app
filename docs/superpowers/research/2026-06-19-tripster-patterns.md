# Tripster UX patterns — competitor reference (live extraction 2026-06-19)
For the Provodnik spine redesign. Tripster = leading RU local-guide marketplace. Adoptable patterns per area + what to do with them.

## 1. Listing cards → tasks 26/27 (listings/search), R-5
- **Format badge overlay** (Групповая / Мини-группа / Индивидуальная / Самостоятельная), bottom-left on image — product type at a glance.
- **Duration + transport overlay** bottom-right ("2,5 часа · пешком").
- Guide **avatar + first name + rating + review count** on the card ("Алексей · 1376 отзывов · 4,87★") — names the human (matches our R-5).
- Price anchored: strikethrough + "1295 ₽ за одного" / "за группу".
- Honest promo labels ("Промо" / "Продвигается" rocket) — RU users expect transparency.
ADOPT: format-badge + duration/transport overlay on our listing cards; keep guide name+face (R-5).

## 2. Guide profile → task 30 (guide-profile), R-3
- **Tagline-as-identity** under the name ("Гид в Кронштадте, офицер запаса") — a 1-line story, NOT a job title.
- **Trust bar:** ★rating · N отзывов · **N посетил** · **N лет на платформе** (tenure + visitor count = strong surrogate trust when reviews are thin).
- Reviews route through experiences; reviewer credibility tag **"Опыт: N экскурсий"**; **"отзыв только после посещения"** lock.
- No CTA on the abstract profile — contact happens at experience/request level.
ADOPT: 1-line identity slot; platform-tenure + "N поездок" trust line; our primary CTA "Оставить запрос этому гиду" (Tripster has none — our reverse model is better here). **Tripster's GAP = no new-guide trust badge → our "Новый гид + Проводник проверил" (R-3) is a real differentiator.**

## 3. Experience detail → task 28 (listing-detail), R-4/R-7
- **Gallery hero**: 1 big + 2×2 mosaic + "N фото" link (Airbnb-style).
- **Sticky sidebar**: large price + primary CTA ("Выбрать дату") + secondary ("Задать вопрос") + **trust micro-copy** ("моментальное бронирование без ожидания").
- **Meta icons row**: transport · language · capacity · format · duration · child-friendly.
- Reviewer "Опыт: N экскурсий"; "отзыв только после посещения".
ADOPT: gallery-mosaic hero; sticky price + primary/secondary CTA + doubt-remover copy (matches R-4); meta-icons row; post-visit review lock.

## 4. Search/discovery → tasks 26/27, R-2
- **Combined date+group pill** ("Любые даты, 1 чел.") — one control, mobile-clean.
- **Рубрики category chips with counts** ("Все 2464 · Со скидкой 272 · …") + "Ещё".
- **Editorial carousels before the algorithmic grid** (curation/trust signal, not pure pay-to-rank).
- Persistent filter bar; explicit sort.
ADOPT: combined date+group pill; trip-type Рубрики chips with counts (matches R-2 facet counts); consider an editorial row.

## 5. Trust + booking flow → cross-cutting
- Post-visit-only reviews + fake-review penalties = table stakes (we must have the review lock).
- Contact shared only after booking (Tripster is book-first/escrow).
- **Provodnik's structural advantage:** reverse-marketplace = dialogue + guide self-selection BEFORE any payment → foreground this in copy as the trust story vs Tripster's "book blind". Our response-SLA line ("гид ответит за ~N ч") is our equivalent of Tripster's instant-book badge (R-8).
