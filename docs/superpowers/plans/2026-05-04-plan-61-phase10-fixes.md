## Task 61-1 [P1] — Replace dead Unsplash image URLs in production listings

**Source:** T069.2 G-F001 through G-F008 (same root)
**Type:** MIGRATION — Supabase Management API (orchestrator executes directly)

**Root cause:**
- `photo-1571079570759-18c2dde62d04` → HTTP 404 at Unsplash upstream (used by 2 listing rows)
- `photo-1520637836862-4d197d17c55a` → HTTP 404 at Unsplash upstream (used by 2 listing rows)

Total broken rows: 4 (2 unique dead photo IDs × 2 rows each). T069.2 reported "2 with broken covers"
because the browser deduplicates image requests by URL — 2 unique 404 URLs appear as 2 broken image
icons in the grid, each triggered by 2 rows.

**Fix — apply via Supabase Management API as a new data-only migration:**

Step 1 — write `provodnik.app/supabase/migrations/20260504000003_fix_listing_images.sql`:

```sql
-- Fix dead Unsplash image URLs from T068 re-seed migration.
-- photo-1571079570759 and photo-1520637836862 return HTTP 404 at source.
-- Replace with confirmed-working IDs from the same migration batch.
-- Note: replacements create visual duplicates until a diversity pass post-launch (non-P1).

UPDATE public.listings
SET image_url = 'https://images.unsplash.com/photo-1543832923-44667a44c804?auto=format&fit=crop&w=1600&h=900&q=80'
WHERE image_url LIKE '%photo-1571079570759%'
  AND status = 'published';

UPDATE public.listings
SET image_url = 'https://images.unsplash.com/photo-1558618666-fcd25c85cd64?auto=format&fit=crop&w=1600&h=900&q=80'
WHERE image_url LIKE '%photo-1520637836862%'
  AND status = 'published';
```

Replacement IDs are confirmed working in the same T068 migration batch (they power the 8/10 listing
cards that loaded without error in the T069.2 audit).

Step 2 — apply via Supabase Management API (project `yjzpshutgmhxizosbeef`).
Token in `codex-ops/.env.local` as `SUPABASE_ACCESS_TOKEN`.

Step 3 — commit the migration file: `fix(data): replace dead Unsplash URLs in production listings (T068 reseed)`

Step 4 — push to `origin/main`.

**Verify:**
- `SELECT count(*) FROM listings WHERE image_url LIKE '%photo-1571079570759%'` returns 0.
