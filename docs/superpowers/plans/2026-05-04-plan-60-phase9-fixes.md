F001, F002, and G-F001 are the **same root cause**: Next.js 16 changed the `<Image priority>` behaviour.
In ≤15, `priority` set both the preload `<link>` AND `fetchpriority="high"` on the rendered `<img>`.
In v16, `priority` only sets the preload link — the attribute on the `<img>` element must be passed
explicitly via the `fetchPriority="high"` prop. Existing audits pass for the preload link but fail on
the DOM attribute check.

## Task 60-1 [P1] — Add `fetchPriority="high"` to all priority `<Image>` calls (Next.js 16 fix)

**Source:** T066.1-F001 + T066.1-F002 + T066.2-G-F001 (same root)
**Type:** DISPATCH — cursor-agent EDIT, 3 files

**Root cause:** Next.js 16 split the `priority` prop into two effects:
1. emits `<link rel="preload" as="image">` in `<head>` (still tied to `priority`)
2. sets `fetchpriority="high"` on the `<img>` (now requires explicit `fetchPriority` prop)

Plan 59 added `priority` everywhere the audit asked for, so the preload links are correct, but the
`<img>` attribute audit fails. The fix is mechanical and identical across three components.

**Files:**
- `src/components/shared/listing-card.tsx` — public `/listings` card
- `src/components/shared/tour-card.tsx` — also touched in Plan 59 (used in `/guides/[slug]` готовые экскурсии grid)
- `src/features/listings/components/public/photo-gallery.tsx` — listing-detail hero
- `src/features/guide/components/guide-listing-card.tsx` — guide cabinet `/guide/listings` card

**Steps (per file):**
1. Find the `<Image>` JSX node that already accepts `priority` (or hard-codes `priority` for the hero/first slot).
2. Add `fetchPriority={priority ? "high" : "auto"}` immediately after the existing `priority` prop.
   For hero/first slots that already pass `priority` unconditionally, pass `fetchPriority="high"` directly.
3. No type changes needed — Next.js's `<Image>` accepts `fetchPriority?: "high" | "low" | "auto"`.
4. Commit: `perf(image): add explicit fetchPriority for Next.js 16 priority Image calls`

**Verify:**
- `bun run typecheck` exits 0.
- DOM at `https://provodnik.app/listings` (1280px): first card `<img>` has `fetchpriority="high"`.
- DOM at `https://provodnik.app/listings/<slug>` (1280px): hero `<img>` has `fetchpriority="high"`.
- DOM at `https://provodnik.app/guide/listings` (logged in as guide, 1280px): first card `<img>` has `fetchpriority="high"`.
