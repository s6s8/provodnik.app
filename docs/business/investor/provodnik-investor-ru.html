Investment Thesis

Почему проект может работать

1. Спрос подтвержден рынком. Крупные игроки уже доказали наличие спроса, доверительных механизмов и готовности пользователей платить за бронирование экскурсий онлайн.

2. У incumbents есть слабое место. Они сильны в listing-first логике, но заметно слабее в custom demand, бюджетных сценариях и user-led group formation.

3. Wedge Provodnik понятен. Пользователь создает запрос по датам, бюджету и формату, другие путешественники могут присоединиться, а гиды отвечают структурированными офферами.

4. GTM дисциплинированный. Запуск с 1–3 регионов и низкорисковых категорий позволяет сначала доказать ликвидность, а не размывать операционную фокусировку.
Investor Lens

Ключевые выводы для инвестора

  • Ниша — это не «еще один агрегатор», а цифровизация слабо структурированного хвоста рынка.
  • Драйвер для гидов — lower take rate и дополнительный канал продаж вне крупных порталов.
  • Драйвер для traveler-side — более низкая цена через сбор группы, гибкие запросы и лучшая прозрачность.
  • Риски известны заранее: доверие, модерация, отмены, возвраты, anti-circumvention, скорость ответа.

Что уже сделано в приложении и что делаем дальше

Проект уже находится не на стадии идеи, а на стадии рабочей MVP-базы: публичный shell, protected workspaces, локальные data contracts, базовые marketplace flows и validated execution queue для следующих релизов.

Done

Что уже сделано

  • Собран public marketing shell и baseline маршрутизация на Next.js 16 App Router.
  • Подняты protected workspace shell и role navigation для traveler, guide и admin.
  • Реализованы traveler request form, локальное хранение запросов, detail flow и comparison офферов.
  • Реализованы guide onboarding, verification intake, request inbox и structured offer composer.
  • Добавлены public guide profile, trust markers, listing discovery, listing detail и booking detail scaffold.
  • Собраны admin queues для проверки гидов и модерации листингов.
  • На main проходят bun run lint, bun run typecheck и bun run build.
Now / Next

Что делаем сейчас

  • Переходим из Release 1 Transaction Foundation в Release 2 Marketplace Quality.
  • Следующий слой ценности: open group joining, favorites, reviews, notifications, disputes и refunds.
