# План: скрыть публичные разделы «Направления» и «Экскурсии»

## Интерпретация

- Скрываем публичный каталог направлений и экскурсий из видимой навигации/главной/SEO.
- Не трогаем рабочие кабинеты: `/guide/listings` и админские листинги остаются, потому что это не публичный раздел каталога, а инструменты гида/админа.
- Убираем мелкий trust/helper-текст под формой запроса на главной.

## Задачи

1. **Навигация**
   - Убрать `/listings` и `/destinations` из публичной, traveller и admin-header навигации.
   - Обновить тест ожиданий в `src/lib/navigation.test.ts`.

2. **Главная**
   - Убрать блок «Популярные направления» из `homepage-shell2-classic`.
   - Упростить загрузку данных в `src/app/(home)/page.tsx`: больше не тянуть каталог направлений ради скрытого блока.
   - Убрать текст под формой в classic hero.
   - Убрать аналогичный helper-текст в новом `HeroConversation`, чтобы текст не вернулся на alternate home/AI flow.

3. **Публичные ссылки на экскурсии**
   - Заменить CTA «Смотреть экскурсии» на сценарий запроса/подбора гида там, где это публичные маркетинговые страницы.

4. **SEO**
   - Убрать `/listings`, `/destinations` и их динамические записи из sitemap, чтобы скрытые разделы не индексировались как активные.

## Проверки

- `bun run typecheck`
- `bun run lint`
- targeted tests around navigation/home/sitemap if available
- production/local smoke главной: убедиться, что нет текста `Бесплатно · без регистрации` и нет visible links на `/listings`/`/destinations` в header/home.
