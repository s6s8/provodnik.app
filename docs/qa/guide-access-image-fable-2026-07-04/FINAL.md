## 2. Комплейнт 3: растянутая картинка на детальной странице гида

### Root cause

`GuideProfileScreen` (коммит #211, 2026-06-21) рендерил **аватар** гида фоном полноэкранного `ImmersiveHero` (`fill`, `sizes="100vw"`, object-cover). Аватар Жюля — 496×618 webp. Реальные замеры до фикса (прод):

- 1280: источник 330×412 → отрисован 1280×632 (апскейл ×3.9 по площади, лицо во весь экран, мыло);
- 375: источник 96×120 → отрисован 375×520 (апскейл ×3.9).

Маленькая пользовательская фотография в принципе не годится в полноэкранный hero — это и есть «растянутая картинка».

### Fix (минимальный)

`src/features/guide/components/public/guide-profile-screen.tsx`:
- ветка «аватар как hero» удалена; типографический hero (инициалы-водяной знак + имя + регион) теперь всегда;
- аватар рендерится в инфо-секции ограниченным кругом: `next/image` c фиксированными `width/height={256}` (2× ретина), CSS `size-28 md:size-32 rounded-full object-cover` — картинка **даунскейлится** (496→256), растяжение исключено конструктивно.

После фикса (локально): 1280 → круг 128×128 из источника 256×319; 375 → круг 112×112. Резко, без апскейла. Console чистая.

Context7: `/vercel/next.js` — `next/image` `width`/`height` задают intrinsic size (браузер резервирует место, рендер-размер контролирует CSS); `fill`+`sizes` — для случаев неизвестного аспекта, что здесь больше не нужно.
