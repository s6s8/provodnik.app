# ADRs

Use ADRs for decisions that should survive individual tasks, branches, and agents.

Add an ADR when you change:
- architecture boundaries
- data ownership
- framework-level patterns
- repo workflow or memory conventions

Naming:
- `0001-short-title.md`
- increment the number

Format:
- Status
- Context
- Decision
- Consequences

Keep ADRs short. Link to code or docs instead of repeating them.
