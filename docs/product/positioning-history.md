## Обоснование Элиста-специфичного copy

Из `research/STRATEGY_DECISIONS_2026-05-26.md` §1–§2:

1. **Команда на месте** — supply-onboarding в РФ требует офлайн-встреч; команда в Элисте делает это за дни, не недели.

2. **Сильный туристический рынок** — Калмыкия лидирует РФ по динамике броней отелей (+48% в 2024, Интерфакс); турпоток за 6 месяцев 2025 почти удвоился.

3. **Целевой сегмент в Элисте — Пары + Семьи/Компании 3+**, не соло. Соло-сегмент тонкий в абсолютных цифрах → Model 1 (Сборная группа) не bootstrapится в Элисте. Killer feature: **удобная форма + низкая комиссия для готовых групп**, не group formation.

4. **Единственный message** — не «для одиночек, пар и компаний» (три message путают пользователя), а один чёткий: «частный гид по Элисте» для семей и компаний.

5. **Festival-driven launch window** — тюльпановый (апрель–май), лотосовый (июль–август). Группы паломников и фестивальщиков → Элиста-специфичный copy точно бил в аудиторию.

## Решение о переходе на универсальный copy

**Дата:** 2026-05-26  
**Commit:** `0273924` (`feat(hero): universal hero copy + env-default city + trust strip (form-epic #3)`)  
**Сообщение-триггер:** форм-эпик message [20] (по ROADMAP.md E-76)

### Причина переопределения

Основная техническая причина — **склонение городов в русском языке**. «По Элисте» работает, но при смене региона (Phase B → Казань, Карелия) «по Казани» / «по Карелии» требует разной логики склонения для каждого нового города. Поддержка в коде становится хрупкой.

**Решение:** убрать город из H1 headline полностью, оставить только в поле «Куда» с env-default.

### Итоговый universal copy (действовал с 2026-05-26; шаблон retired)

> Шаблон subtitle с публичным процентом retired 2026-05-27, когда ставка комиссии стала внутренней информацией и перестала выводиться в публичном UI.

**H1:**
```
Частный гид — для вашей семьи или компании
```

**Subtitle (интерполированный):**
```
Опишите запрос — местные гиды ответят за пару часов.
Мы берём {COMMISSION_PCT}%, остальное идёт гиду напрямую.
```
_(историческая интерполяция; текущий канон комиссии см. в `docs/product/PRD.md` §15 и KODEX «Три модели продукта»)_

**Поле «Куда» (default):**
```
NEXT_PUBLIC_PHASE_A_CITY env → fallback «Элиста»
```

### Преимущества universal-подхода

- Phase A→B переход требует только ротации `NEXT_PUBLIC_PHASE_A_CITY`, без code redeploy.
- Нет проблем склонения при смене региона.
- H1 остаётся ценностным заявлением («для вашей семьи или компании»), а не географическим тегом.
- Элиста остаётся «вшитой» как fallback — если env не задан, форма по-прежнему дефолтится на Элисту.
