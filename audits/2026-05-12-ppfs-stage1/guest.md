## Cross-cutting / themed observations

- **Encoding bug, P1:** Two `/listings` rows (`moscow-boulevards-and-hidden-yards`, `moscow-hidden-history-tour`) — and the single seed guide's bio — render as mojibake. The pattern (every Cyrillic byte becomes `?`) indicates the source bytes are not valid UTF-8; root cause is at the data layer (migration / seed insert / encoding pragma), not the renderer. Fix candidates: (a) re-seed the affected rows from a known UTF-8 source; (b) audit `supabase/migrations/*seed*.sql` for non-UTF-8 inputs; (c) add a one-off DB normaliser run.
- **Anti-disintermediation surface — relevant for ticket #4 of epic 393:** Listing detail already shows "Контакты гида откроются после принятия предложения" and "0% комиссии" copy — anti-dez framing already present at the touchpoint. Anti-dez ticket should treat this as the baseline and extend (messaging filters, profile contact masking, retention screen on close-attempt).
- **Auth-flow edge cases:** `/auth/update-password` (no token) and `/auth/confirm` (no token) both render or redirect silently without surfacing what happened. Recurring theme: error states swallow signal.
- **Empty-state polish:** `/guides`, `/guides/[slug]`, `/listings` cards all show `★ 0 · 0 отзывов` or `0.0 / 5` when there are no reviews. Should be hidden or replaced with `Без отзывов` to avoid implying a zero-star guide.
- **Stub-vs-reality drift:** `/tours` is in the registry but 404s. Audit registry needs a "deprecated / not-yet-built" annotation, or the route map needs realignment.
